# NOK图片飞书发送功能实施总结

## 功能概述

实现了"NOK图片飞书发送"功能，允许用户在确认图片为NOK时，选择是否将图片发送到指定的飞书群。

## 实施内容

### 1. 配置文件修改

**文件**: `config.json`

添加了飞书设置配置项：
```json
"feishu_settings": {
    "nok_send_enabled": false,
    "chat_id": ""
}
```

### 2. 配置处理器扩展

**文件**: `config_handler.py`

添加了以下方法：
- `get_feishu_settings()` - 获取飞书设置
- `update_feishu_settings(nok_send_enabled, chat_id)` - 更新飞书设置

### 3. 后端API扩展

**文件**: `app.py`

添加了3个新的API接口：

1. **GET /api/settings** - 获取应用设置
   - 返回飞书发送开关状态和群组ID

2. **POST /api/settings** - 保存应用设置
   - 接收并保存飞书发送开关状态和群组ID

3. **POST /api/feishu/send** - 发送图片到飞书群
   - 接收图片信息和群组ID
   - 上传图片到飞书获取image_key
   - 构造富文本消息（包含项目文件夹、图片名称、操作时间、操作用户、图片）
   - 发送消息到指定飞书群

添加了设置页面路由：
- **GET /settings** - 设置页面

### 4. 前端页面

#### 设置页面

**文件**: 
- `templates/settings.html` - 设置页面HTML
- `static/js/settings.js` - 设置页面逻辑

**功能**:
- NOK图片飞书发送开关（checkbox）
- 飞书群ID输入框（text input）
- 保存和重置按钮
- 支持深色/浅色主题
- 表单验证

#### 主页面修改

**文件**: `templates/index.html`

在侧边栏导航链接区域添加了"设置"按钮。

#### 主页面逻辑修改

**文件**: `static/js/main.js`

修改了NOK分类逻辑：
1. 页面加载时获取飞书设置
2. 当用户点击NOK按钮时：
   - 如果飞书发送开关关闭，直接执行NOK分类
   - 如果飞书发送开关开启，弹出确认对话框："是否发送此图片到飞书群？"
     - 点击确认：执行NOK分类 + 发送飞书消息
     - 点击取消：仅执行NOK分类，不发送飞书消息

## 交互流程

```
用户点击NOK按钮
    ↓
检查飞书发送开关
    ↓
[关闭] → 直接分类为NOK
    ↓
[开启] → 弹出确认对话框："是否发送此图片到飞书群？"
    ↓
[取消] → 仅执行NOK分类
    ↓
[确认] → 执行NOK分类 + 发送飞书消息
    ↓
显示操作结果通知
```

## 飞书消息格式

发送的富文本消息包含：
- 标题：NOK图片通知
- 内容：
  - 项目文件夹名称
  - 图片文件名
  - 操作时间
  - 操作用户
  - NOK图片（内嵌显示）

## 使用说明

### 配置步骤

1. 点击主页侧边栏的"设置"按钮
2. 打开"NOK图片飞书发送"开关
3. 输入飞书群的chat_id（格式如: oc_xxxxx）
4. 点击"保存设置"按钮

### 使用流程

1. 在主页面审查图片
2. 当确认图片为NOK时，点击"NOK"按钮
3. 如果飞书发送功能已启用，会弹出确认对话框
4. 选择"确定"发送到飞书群，或选择"取消"仅分类不发送
5. 图片会被移动到NOK文件夹
6. 如果选择发送，图片和相关信息会被发送到指定飞书群

## 技术特性

- **错误处理**: 飞书发送失败不影响NOK分类操作
- **配置持久化**: 设置保存在config.json文件中
- **用户体验**: 使用原生confirm对话框，简洁直观
- **主题支持**: 设置页面支持深色/浅色主题切换
- **表单验证**: 启用飞书发送时必须填写群组ID
